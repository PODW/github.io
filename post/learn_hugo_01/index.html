Hugo 學習日誌01

 2020-12-29 Hugo 網站架設 

本站是透過Hugo這個靜態網站生成器製作而成的
網路上還有其他的靜態網站生成器
而我會選擇用Hugo的原因就是因為它安裝簡單生成快速
我這裡只記錄怎麼用Hugo製作靜態網站跟我遇到的問題
其他更詳細的內容請再上網搜尋

(一)安裝 Hugo

說是安裝 Hugo 其實只需要一個 hugo.exe 就可以執行
所以只要去  Hugo官網下載  符合你電腦環境的執行檔就好

我是在 Windows10 x64 系統環境下所以是載

hugo_extended_(0.79.1)1_Windows-64bit.zip

將檔案解壓縮後就會得到一個 hugo.exe
其實現在就可以執行 Hugo 了，但是只能在目前解壓縮出來的目錄下
為了之後執行方便，我建議修改環境變數 2
讓 Windows PowerShell 或 Cmd 不管在哪都可執行 Hugo

照著圖片操作最後按下確認就完成 Hugo 的安裝
這時候可以打開 Cmd 來確認，執行指令 hugo version
可以看到版號就代表成功了

(二)創建一網站

在確認Hugo可以執行後，下一步就是創建一個網站
在你想要放網站檔案的位置開啟Windows PowerShell
執行指令hugo new site (Name) ，Name 為你網站要取的名稱

執行完畫面會如上圖，同時也會建出一個名稱 (Name) 的資料夾
你會注意到 Windows PowerShell 其實也會指示你接下來要怎麼做
所以我們就依照它的指示先去 https://themes.gohugo.io/ 去選擇我們主題
每個主題都可以點進去看，選擇自己喜歡的主題去下載
按下 Download 後，會到這個主題的 GitHub3 頁面
直接點選 Download  ZIP 下載就好

下載完將 ZIP 檔名後面的-master刪除
再放到剛創建 (Name) 的資料夾裡的 themes 資料夾下並解壓縮

解開後你通常可以看到一個叫 exampleSite 的資料夾
將裡面的檔案複製下來，回到 (Name) 的根目錄將它們貼上
理論上這樣網站就已經有一個雛型了
這時在 (Name) 的根目錄開啟 Windows PowerShell
執行指令hugo server後，你就可以在 http://localhost:1313/看到目前在本機運行的網站

可能遇到的問題

  1. hugo servver執行時出現Error，而不會出現上面的圖
  • 我遇到的問題之一，他顯示說我的主題太舊現在不支援，後來換一個主題就好了，所以在選主題時要注意它是否太久沒更新

  1. 連到 http://localhost:1313/ 顯示不出網站來
  • 我遇到的問題之一，我一開始用 Google chrome 會無法顯示，換個瀏覽器(Firefox, Edge)就好了

  1. 其他問題
  • 因為每個主題都有不同的格式，所以可能還有其他要求才能執行網站，下載主題前可以先看主題下方的描述或解壓縮完的ReadMe的檔案，通常都會有作者的提示怎麼使用他的主題

(三)創建文章

確認網站可以運作後，接下就是開始改造這個網站
但要開始改造網站，你就需要一個文字編輯器
而網路上有很多不同的文字編輯器，基本上只要能正常開啟.md4 檔的都可以
我個人使用的是 Visual studio code 這個編輯器
會選用這個編輯器的原因是它可以在編輯檔案時呼叫 Windows PowerShell，同時也有許多模組可以使用
這樣可以不用一直切換視窗去下指令，增加編輯效率
你可以到 Visual studio code 官網 去下載
安裝完打開後應該會類似下圖

如果你不習慣用英文介面，Visual studio code 有繁體中文的模組
照著下圖操作你就會看到繁體中文的模組，點擊安裝後 Visual studio code 就會變成繁體介面了

接下來才是重點，為了編輯 Hugo 產生文章跟網頁設定
請安裝下面圖示的模組，為了編輯.md跟.toml檔時能更好閱讀建議安裝

做完上述動作後，我們的工作環境就準備好了
接下來就是用 Visual studio code 開啟之前創建的 (Name) 資料夾
你可以點選上方的終端機，也可以用快捷鍵CTRL+~來呼叫 Windows PowerShell
先執行 hugo server 來讓網站運行
再來執行指令 hugo new post/(Name2).md，Name2 為你的文章標題
執行完後你會發現你的網站多了一篇空白文章，而檔案位置就在 post 資料夾裡
而裡面的其他檔案就是其它文章，你可以參考範例看是怎麼寫的並跟著改寫
我的第一篇文章也有 Markdown 的一些用法
只要不用CTRL+C來中止網站，你對檔案的的改動就會直接反應在網站上
你可以一邊開著網頁一邊修改你的文章，查看是否有如你預期更改文章

  1. 看目前Hugo最新是哪一版。 ↩︎

  2. (Windows)作業系統會依照PATH環境變數中所設定的路徑順序，依序尋找各路徑可以執行命令的執行檔。 ↩︎

  3. 簡單來說就是儲存紀錄各種程式碼和檔案的網站，建議沒有帳戶可以去 GitHub 註冊，方便管理你之後網站的檔案，同時也能讓你的網站上線。 ↩︎

  4. 是一種叫 Markdown 格式撰寫的文件檔，詳情請看 維基 ↩︎
